# 모델·에포트 라우팅 (Model & Effort Routing)

## 언어 정책 (필수)

- 내부 분석, 계획, 문제 해결, 도구 호출, 검색어 작성 등 모든 사고 및 작업 과정은 **영어를 우선**한다.
- 사용자에게 전달하는 진행 업데이트와 최종 답변은 **한국어를 우선**한다.
- 코드, 명령어, API, 라이브러리, 모델명, 파일명과 기타 기술 용어는 정확성과 업계 관례를 위해 영어로 작성해도 된다.
- 사용자가 특정 언어를 명시하면 해당 요청을 우선한다.

## RTK/Caveman 상시 적용 (필수)

- 런타임에 `caveman` skill이 설치되어 있으면, 매 세션의 첫 응답부터 별도 호출 없이 반드시 Caveman `full` 응답 형식을 적용한다.
- 적용 상태는 모든 턴과 세션에서 유지한다. 사용자가 `stop caveman` 또는 `normal mode`를 명시한 경우에만 해제한다.
- 진행 업데이트와 최종 답변 모두에 적용한다. 불필요한 인사, 도구 호출 서술, 장식 표, 장황한 로그를 제거한다.
- 사용자의 주 언어를 유지하고 코드, 명령어, API, 파일 경로, 정확한 오류 문자열은 변경하지 않는다.
- 보안 경고, 비가역 작업 확인, 압축으로 의미가 불명확해지는 절차만 명료성을 위해 일시적으로 확장하고, 이후 즉시 Caveman 형식으로 복귀한다.

## 목적과 기본 원칙

- 총비용과 지연은 대략 `모델 등급 × 에포트 × 에이전트 수`에 비례한다.
- 세 다이얼을 한꺼번에 올리지 말고, 필요한 다이얼만 한 단계씩 올린다.
- 기본값은 **단일 에이전트 · GPT-5.6-Terra · medium**이다.
- Sol, max, ultra, 멀티에이전트를 상시 사용하지 않는다.
- 시스템·플랫폼·사용자 지침과 더 가까운 경로의 `AGENTS.md`가 이 문서보다 우선한다.

## 모델 라인업

| 모델 | 위치 | 주 용도 |
|---|---|---|
| `gpt-5.6-sol` (GPT-5.6-Sol) | 프론티어 | 대형 설계, 어려운 근본 원인 분석, 적대적 리뷰, XL 핵심 난제 |
| `gpt-5.6-terra` (GPT-5.6-Terra) | 밸런스 | 일반 구현, 기능 개발, 대부분의 프로덕션 코딩 |
| `gpt-5.6-luna` (GPT-5.6-Luna) | 고속·저비용 | 기계적 조사, 검색, 분류, 단순 편집, 작은 수정 |

### Sol

- L/XL 아키텍처와 계획, 복잡한 근본 원인 분석, 대형 리팩터링의 핵심 설계, 적대적 PR 리뷰에 사용한다.
- Terra로 해결되지 않은 핵심 난제에 제한적으로 사용한다.
- 단순 편집, 검색, 목록화, 익숙한 기능 구현에는 사용하지 않는다.

### Terra

- 일반 기능 구현, 여러 파일 수정, 테스트 작성, 버그 수정, 중간 규모 리팩터링에 사용한다.
- 별도 근거가 없으면 메인 구현 세션은 Terra·medium을 선택한다.

### Luna

- 파일·심볼 검색, 저장소 구조 조사, 반복 편집, 테스트 실패 분류, S급 작업에 사용한다.
- 기계적 서브에이전트 작업의 기본 모델이다.
- 복잡한 아키텍처 판정이나 최종 종합을 Luna 결과만으로 확정하지 않는다.

## 에포트 레벨

| 에포트 | 사용 기준 |
|---|---|
| low | 검색, 분류, 단순 편집, 명확한 S급 작업 |
| medium | 일반 구현의 기본값 |
| high | 복잡한 분석, 계획, 코드 리뷰 |
| xhigh | 대형 리팩터링, 깊은 디버깅, L/XL 작업 |
| max | xhigh로 부족함이 확인된 최난도 단발 문제 |
| ultra | 최대 추론과 자동 작업 위임이 모두 필요한 제한적 작업 |

- Luna는 low~max를 사용한다.
- Sol과 Terra는 low~ultra를 사용할 수 있다.
- `ultra`는 자동 위임까지 포함하므로 단순히 더 깊게 생각하기 위한 모드로 사용하지 않는다.
- 상향 순서는 `현재 모델에서 effort 한 단계 상승 → 문제 분해 → 독립 조사 병렬화 → Luna에서 Terra로 또는 Terra에서 Sol로 모델 상승 → max → 필요 시 ultra`이다.

## 복잡도 티어

| 티어 | 기준 |
|---|---|
| S | 단일 파일, 익숙한 패턴, 작은 버그 수정, 단순 정리 |
| M | 기능 단위, 설계가 비교적 명확함, 여러 파일 수정 |
| L | 크로스스택, 재설계, 대형 리팩터링, 감사 |
| XL | 신규 도메인, 탐색적 설계, 불확실성과 직렬 임계경로가 큼 |

파일 수만으로 티어를 정하지 않는다. 요구사항 불확실성, 파급 범위, 데이터·보안 위험, 되돌리기 비용과 검증 난도를 함께 고려한다.

## 티어 × 단계 매트릭스

`●●●`는 멀티에이전트가 정당할 수 있음을 뜻한다. 상위 지침이나 사용자가 허용한 경우에만 사용한다.

| 티어 | ① 계획 | ② 계획 검토 | ③ 구현 | ④ PR 리뷰 |
|---|---|---|---|---|
| S | 생략 또는 Luna·low | 셀프 검토 | Luna·low/medium | Terra·medium |
| M | Terra·high | Terra·high | Terra·medium | Terra·high |
| L | Sol·high/xhigh | Sol·xhigh ●●● | Terra·medium/high | Sol·xhigh |
| XL | Sol·xhigh | Sol·xhigh ●●● | Terra·high, 핵심만 Sol·xhigh | Sol·xhigh |

예외:

- S 작업이라도 보안, 결제, 권한, 데이터 손실 위험이 있으면 Terra·high 이상을 사용한다.
- L 작업이라도 구현이 기계적이면 구현 단계는 Terra·medium으로 내린다.
- 저장소 전수 감사, 문서↔코드 검증 스윕은 병렬화가 정당할 수 있다.
- 모델이나 effort를 높였다는 이유로 테스트와 검증을 생략하지 않는다.

## 메인 세션 체크 (필수)

새 작업을 받으면 첫 진행 응답에서 다음을 짧게 명시한다.

1. 복잡도 티어: S / M / L / XL
2. 단계: 계획 / 계획 검토 / 구현 / PR 리뷰
3. 권장 모델과 effort
4. 단일 또는 멀티에이전트 여부
5. 현재 세션 설정이 권장값과 다르면 전환 안내

예시:

> 판정: M · 구현 단계. Terra·medium과 단일 에이전트가 권장됩니다.

- 현재 설정이 과하면 `/model`과 `/effort`로 하향 전환을 권고한다.
- 현재 설정이 부족하면 작업 시작 전에 `/model`과 `/effort`로 상향 전환을 권고한다.
- 에이전트는 메인 세션 설정을 스스로 변경했다고 주장하지 않는다. 안내만 한다.
- 사용자가 전환하지 않고 계속 진행하면 현재 설정으로 작업한다.

## 서브에이전트 라우팅

서브에이전트는 작업이 실제로 독립적이고 병렬 실행의 이득이 조정 비용보다 클 때만 사용한다.

### 기계적 조사

- 예: 파일 존재 확인, `rg` 검색, 참조 목록화, 테스트 실패 분류, 반복 패턴 조사.
- 권장: Luna·low.
- 출력 범위를 좁게 지정하고 최종 판단은 메인 에이전트가 수행한다.

### 일반 구현

- 예: 독립 패키지 수정, 서로 겹치지 않는 파일 작업, 분리된 테스트 작성.
- 권장: Terra·medium.
- 동일 파일을 여러 에이전트가 동시에 수정하지 않게 한다.

### 판단·비평

- 예: 근본 원인 판정, 아키텍처 비평, 적대적 리뷰, 보안·성능·접근성 렌즈.
- 권장: Terra·high 또는 Sol·high/xhigh.
- 근거와 불확실성을 보고하게 하고 최종 종합은 메인 에이전트가 수행한다.

## 멀티에이전트 규칙

- 하위 작업이 서로 독립적이고, 병렬화로 실제 시간이 줄며, 결과를 명확히 병합할 수 있을 때만 사용한다.
- 사용자가 요청했거나 상위 지침이 허용한 경우에만 사용한다.
- 저장소 전수 감사, 독립 리뷰 렌즈, 문서↔코드 검증, 여러 패키지의 독립 조사에 적합하다.
- 단일 파일 수정, 순차 의존 작업, 작은 버그 수정, 동일 문제 중복 할당에는 사용하지 않는다.
- 동시 활성 에이전트는 환경 제한을 따르며, 별도 제한이 없다면 최대 4개로 제한한다.

## 실전 요약

| 작업 | 권장 설정 |
|---|---|
| 오타, 포맷, 단순 설정 변경 | Luna·low |
| 파일 검색, 목록화, 실패 분류 | Luna·low |
| 작은 버그 수정 | Luna·medium 또는 Terra·medium |
| 일반 기능 구현 | Terra·medium |
| 여러 파일 리팩터링 | Terra·medium/high |
| M급 계획과 리뷰 | Terra·high |
| L급 설계와 계획 | Sol·high/xhigh |
| 어려운 근본 원인 분석 | Sol·xhigh |
| 적대적 PR 리뷰 | Sol·high/xhigh |
| 최난도 단발 문제 | Sol·max |
| 최대 추론과 자동 위임이 모두 필요 | Sol·ultra |

## 금지 패턴

- 모든 작업을 Sol 또는 xhigh/max/ultra로 시작하지 않는다.
- 단순 검색을 고비용 모델에 맡기지 않는다.
- 복잡한 최종 판정을 Luna 결과만으로 확정하지 않는다.
- 멀티에이전트를 작업 규모의 과시 수단으로 사용하지 않는다.
- 같은 파일을 여러 에이전트가 동시에 수정하게 하지 않는다.
- `ultra`를 일반적인 고성능 모드처럼 상시 사용하지 않는다.
